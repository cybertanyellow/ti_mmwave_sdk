    46    /*!
    47     *  ======== Hwi ========
    48     *  Hardware Interrupt Support Module.
    49     *
    50     *  This Hwi module provides ARM cortex-A Generic Interrupt Controller(GIC) v2.0
    51     *  specific implementations and extensions of the APIs defined in
    52     *  {@link ti.sysbios.interfaces.IHwi IHwi}.
    53     *
    54     *  The GIC is logically partitioned into 2 blocks, the Distributor block
    55     *  and CPU interface block. The Distributor block interfaces with the interrupt
    56     *  sources, prioritizes interrupts and distributes them to the CPU interface
    57     *  block. The CPU interface block connects to the processors in the system
    58     *  and is responsible for priority masking and preemption handling for
    59     *  the processor it is connected to.
    60     *
    61     *  The GIC can implement up to 8 CPU interfaces with each CPU interface
    62     *  being able to see up to 1020 interrupts. The GIC assigns interrupt ID
    63     *  numbers 0-1019 as follows:
    64     *   - Interrupt numbers 0-31 are used for interrupts private to a
    65     *     CPU interface. These private interrupts are banked in the Distributor.
    66     *   - Banked interrupt number 0-15 are used for Software Generated Interrupts
    67     *     or SGIs.
    68     *   - Banked interrupt number 16-31 are used for Private Peripheral Interrupts
    69     *     or PPIs.
    70     *   - Interrupt numbers 32-1019 are used for Shared Peripheral Interrupts
    71     *     or SPIs.
    72     *   - Interrupt numbers 1020-1023 are reserved.
    73     *
    74     *  @a(NOTE)
    75     *  In the SoC Technical Reference Manual, the MPU IRQs 0 to N map to
    76     *  GIC interrupt numbers 32 to (N+32) where (N+1) is the total number of
    77     *  Shared Peripheral Interrupts implemented.
    78     *  For instance on OMAP5430, MPU IRQ 0 to 159 maps to GIC interrupt number
    79     *  32 to 191.
    80     *
    81     *  @a(INTERRUPT GROUPING)
    82     *  GIC allows configuring an interrupt as a Group 0 or a Group 1 interrupt.
    83     *  Group 0 interrupts are Secure interrupts and Group 1 interrupts are
    84     *  Non-secure interrupts.
    85     *
    86     *  If {@link #enableSecureMode} is set to true, this module supports both
    87     *  Group0 and Group 1 interrupts. Group 0 interrupts are delivered to the CPU
    88     *  using FIQ signal whereas Group 1 interrupts are delivered using IRQ signal.
    89     *
    90     *  If {@link #enableSecureMode} is set to false, this module only supports
    91     *  Group 1 interrupts which are delivered to the target CPU using IRQ signal.
    92     *
    93     *  @a(INTERRUPT PRIORITIES)
    94     *  In general GIC supports priority values 0 thru 255.
    95     *
    96     *  In practice valid priority values depend on the particular device used,
    97     *  security mode and the Binary Point Register (see {@link #BPR} and
    98     *  {@link #ABPR}) value.
    99     *
   100     *  The device implementation and security mode decide the number of priority
   101     *  bits that are implemented (see {@link #NUM_PRIORITY_BITS}). Group 0
   102     *  interrupts always implement one more priority bit than Group 1 interrupts.
   103     *
   104     *  In GIC, interrupts with lower priority numbers have higher priority.
   105     *
   106     *  @a(NOTE)
   107     *  In this Hwi module implementation, the instance config parameter value
   108     *  {@link #MaskingOption_LOWER} is equivalent to {@link #MaskingOption_SELF}.
   109     *  Statically configuring a Hwi object's {@link #Params.maskSetting} to
   110     *  {@link #MaskingOption_LOWER} will result in the generation of a benign
   111     *  build warning. Dynamic usages of {@link #MaskingOption_LOWER} will be
   112     *  silently converted to {@link #MaskingOption_SELF}.
   113     *
   114     *  @a(ZERO LATENCY INTERRUPTS)
   115     *  On Keystone2 devices, this module supports zero-latency interrupts. A
   116     *  zero-latency interrupt does not go through the SYS/BIOS dispatcher and
   117     *  thus has a faster response time. Since zero-latency interrupts bypass the
   118     *  dispatcher, their handler function cannot call any SYS/BIOS APIs.
   119     *
   120     *  This module implements zero-latency interrupts by forwarding the interrupt
   121     *  to the target CPU using FIQ signal. Therefore, in order to configure an
   122     *  interrupt as a zero-latency interrupt, the Hwi type needs to be changed
   123     *  to FIQ when creating or constructing a Hwi.
   124     *
   125     *  Example showing how to create a zero-latency Hwi:
   126     *  @p(code)
   127     *  Void main(Void)
   128     *  {
   129     *      Hwi_Params hwiParams;
   130     *      Hwi_Params_init(&hwiParams);
   131     *      // Default Hwi type is IRQ
   132     *      hwiParams.type = Hwi_Type_FIQ;
   133     *      Hwi_create(INT_NUM_FIQ, myIsrFIQ, &hwiParams, NULL);
   134     *      ...
   135     *  }
   136     *  @p
   137     *
   138     *  FIQs run on their own stack. See {@link #fiqStack} and {@link #fiqStackSize}
   139     *  for more info on how to control the FIQ stack.
   140     *
   141     *  @a(NOTE)
   142     *  This module is written for GIC v2.0, however it is backwards compatible
   143     *  with GIC v1.0
   144     */
   145

   158         *  This field specifies the MPU's security mode. The MPU's security mode

   201        /*!
   202         *  ======== BPR ========
   203         *  GIC Binary Point Register value
   204         *
   205         *  Defines the point at which the priority value fields split into
   206         *  two parts, the group priority field and the sub-priority field.
   207         *  When running in SECURE mode, BPR applies to Group 0 interrupts
   208         *  and when running in NON-SECURE mode, BPR applies to Group 1
   209         *  interrupts.
   210         *
   211         *  The group priority field determines interrupt preemption in case
   212         *  of nested interrupts whereas sub-priority field is used to determine
   213         *  priority within a group when multiple interrrupts belonging to the
   214         *  same group are pending.
   215         *
   216         *  Valid BPR values are from 0-7 with the minimum value supported being
   217         *  implementation defined and in the range 0-3.
   218         *
   219         *  @p(code)
   220         *   -------------------------------------------------------
   221         *  | BPR value | Group priority field | Sub-priority field |
   222         *   -------------------------------------------------------
   223         *  |     0     |         [7:1]        |         [0]        |
   224         *  |     1     |         [7:2]        |        [1:0]       |
   225         *  |     2     |         [7:3]        |        [2:0]       |
   226         *  |     3     |         [7:4]        |        [3:0]       |
   227         *  |     4     |         [7:5]        |        [4:0]       |
   228         *  |     5     |         [7:6]        |        [5:0]       |
   229         *  |     6     |          [7]         |        [6:0]       |
   230         *  |     7     |     No preemption    |        [7:0]       |
   231         *   -------------------------------------------------------
   232         *  @p
   233         *
   234         */
   235        config UInt BPR;

   262        /*!
   263         *  ======== IntAffinity ========
   264         *  Interrupt affinity type. Stores the hierarchical address (composed
   265         *  of different affinity levels) that uniquely identifies the core or
   266         *  processing element (PE) an interrupt is routed to.
   267         *
   268         *  The routingMode field determines whether an interrupt is routed to
   269         *  all nodes or a node identified by the affinity fields. This field
   270         *  takes the value Hwi_RoutingMode_ANY and Hwi_RoutingMode_NODE.
   271         *
   272         *  If {@link ti.sysbios.BIOS#smpEnabled BIOS.smpEnabled} is true then,
   273         *  all interrupts are forwarded to core 0 by default.
   274         *  If {@link ti.sysbios.BIOS#smpEnabled BIOS.smpEnabled} is false then,
   275         *  all interrupts are forwarded to all participating nodes.
   276         *
   277         *  @a(constraints)
   278         *  This config param is ignored if
   279         *  {@link ti.sysbios.family.arm.v8a.Core#bootMaster Core.bootMaster} is
   280         *  set to false.
   281         */

   283            UInt8       aff0;        /*! Affinity level 0 - CoreId within cluster */
   284            UInt8       aff1;        /*! Affinity level 1 - Cluster Id */
   285            RoutingMode routingMode; /*! Routing Mode - ALL or particular NODE */
   286        };

   712        /*!
   713         *  ======== disable ========
   714         *  Globally disable interrupts.
   715         *
   716         *  Hwi_disable globally disables hardware interrupts and returns an
   717         *  opaque key indicating whether interrupts were globally enabled or
   718         *  disabled on entry to Hwi_disable().
   719         *  The actual value of the key is target/device specific and is meant
   720         *  to be passed to Hwi_restore().
   721         *
   722         *  Call Hwi_disable before a portion of a function that needs
   723         *  to run without interruption. When critical processing is complete, call
   724         *  Hwi_restore or Hwi_enable to reenable hardware interrupts.
   725         *
   726         *  Servicing of interrupts that occur while interrupts are disabled is
   727         *  postponed until interrupts are reenabled. However, if the same type
   728         *  of interrupt occurs several times while interrupts are disabled,
   729         *  the interrupt's function is executed only once when interrupts are
   730         *  reenabled.
   731         *
   732         *  A context switch can occur when calling Hwi_enable or Hwi_restore if
   733         *  an enabled interrupt occurred while interrupts are disabled.
   734         *
   735         *  Hwi_disable may be called from main(). However, since Hwi interrupts
   736         *  are already disabled in main(), such a call has no effect.
   737         *
   738         *  @a(NOTE)
   739         *  Disables only IRQ interrupts
   740         *
   741         *  @a(constraints)
   742         *  If a Task switching API such as
   743         *  {@link ti.sysbios.knl.Semaphore#pend Semaphore_pend()},
   744         *  {@link ti.sysbios.knl.Semaphore#post Semaphore_post()},
   745         *  {@link ti.sysbios.knl.Task#sleep Task_sleep()}, or
   746         *  {@link ti.sysbios.knl.Task#yield Task_yield()}
   747         *  is invoked which results in a context switch while
   748         *  interrupts are disabled, an embedded call to
   749         *  {@link #enable Hwi_enable} occurs
   750         *  on the way to the new thread context which unconditionally re-enables
   751         *  interrupts. Interrupts will remain enabled until a subsequent
   752         *  {@link #disable Hwi_disable}
   753         *  invocation.
   754         *
   755         *  Swis always run with interrupts enabled.
   756         *  See {@link ti.sysbios.knl.Swi#post Swi_post()} for a discussion Swis and
   757         *  interrupts.
   758         *
   759         *  @b(returns)     opaque key for use by Hwi_restore()
   760         */
   761        @Macro
